# Encapsulation in Java

## 📌 What is Encapsulation?

Encapsulation is one of the four pillars of **Object-Oriented Programming (OOP)**.

**Definition:**

> Encapsulation is the process of **binding data (variables)** and **methods (functions)** together into a single unit (class) while restricting direct access to the data.

In Java, encapsulation is achieved by:

- Making variables **private**
- Accessing them using **Getter** and **Setter** methods

---

# Why Do We Need Encapsulation?

Without encapsulation, anyone can directly access and modify the object's data.

### Example (Without Encapsulation)

```java
class Human {

    int age;
    String name;
}

public class Demo {

    public static void main(String[] args) {

        Human obj = new Human();

        obj.age = 18;
        obj.name = "Anshika";

        System.out.println(obj.name);
    }
}
```

### ❌ Problems

- Anyone can modify the variables.
- No validation is possible.
- Poor security.
- Violates Data Hiding.

---

# Making Variables Private

```java
class Human {

    private int age;
    private String name;
}
```

Now the variables cannot be accessed directly.

```java
Human obj = new Human();

obj.age = 20;      // Compile-Time Error
obj.name = "John"; // Compile-Time Error
```

### Why?

Because **private members are accessible only inside the same class.**

---

# Accessing Private Variables

Private variables are accessed using methods.

There are two special methods:

- Getter
- Setter

---

# Getter Method

A **Getter** returns the value of a private variable.

```java
class Human {

    private int age = 18;
    private String name = "Anshika";

    public int getAge() {
        return age;
    }

    public String getName() {
        return name;
    }
}
```

### Example

```java
public class Demo {

    public static void main(String[] args) {

        Human obj = new Human();

        System.out.println(obj.getName());
        System.out.println(obj.getAge());
    }
}
```

### Output

```
Anshika
18
```

---

# Setter Method

A **Setter** updates the value of a private variable.

```java
class Human {

    private int age;
    private String name;

    public void setAge(int age) {
        this.age = age;
    }

    public void setName(String name) {
        this.name = name;
    }
}
```

---

# Using Getter and Setter Together

```java
class Human {

    private int age;
    private String name;

    public void setAge(int age) {
        this.age = age;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public String getName() {
        return name;
    }
}
```

### Main Method

```java
public class Demo {

    public static void main(String[] args) {

        Human obj = new Human();

        obj.setAge(20);
        obj.setName("Anshika");

        System.out.println(obj.getName() + " : " + obj.getAge());
    }
}
```

### Output

```
Anshika : 20
```

---

# Understanding `this` Keyword

```java
public void setAge(int age) {
    this.age = age;
}
```

Here,

- `age` → Method parameter
- `this.age` → Instance variable of the current object

The `this` keyword refers to the **current object's instance variable**.

---

# Benefits of Encapsulation

- Protects data from unauthorized access.
- Supports Data Hiding.
- Allows validation before updating data.
- Improves security.
- Makes code easier to maintain.
- Improves code reusability.

---

# Real-Life Example

Think of an **ATM Machine**.

- The account balance is **private**.
- You cannot modify it directly.
- You interact with it through methods like:
  - Deposit
  - Withdraw
  - Check Balance

This is a real-world example of **Encapsulation**.

# Interview Questions

### 1. What is Encapsulation?

Binding data and methods into a single unit while restricting direct access to the data.

---

### 2. How is Encapsulation achieved in Java?

- Private variables
- Public Getter methods
- Public Setter methods

---

### 3. Why do we use private variables?

To prevent direct access and protect the object's data.

---

### 4. What is a Getter?

A method used to **read** the value of a private variable.

---

### 5. What is a Setter?

A method used to **modify** the value of a private variable.

---

### 6. Can private variables be accessed outside the class?

No.

They can only be accessed using public Getter and Setter methods.

# Key Takeaways

- Encapsulation is one of the four pillars of OOP.
- Declare variables as **private**.
- Use **Getter** methods to read data.
- Use **Setter** methods to update data.
- Encapsulation improves **security**, **maintainability**, and **code quality**.

> **Encapsulation = Data Hiding + Controlled Access**
