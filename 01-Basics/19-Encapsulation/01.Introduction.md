# 🤔 Why Do We Need Encapsulation?

Imagine a Bank Account.

```text
Bank Account

↓

Balance = ₹50,000
```

If everyone could directly change the balance,

```java
account.balance = -50000;
```

or

```java
account.balance = 999999999;
```

the application would become unreliable.

We need a mechanism to protect data from unauthorized access.

That mechanism is **Encapsulation**.

---

# 📖 What is Encapsulation?

**Encapsulation** is the process of wrapping data (variables) and methods (functions) into a single unit (class) while restricting direct access to the data.

Instead of allowing direct modification, access is provided through controlled methods.

---

# 🧠 Definition

> **Encapsulation is the technique of binding data and the methods that operate on that data into a single unit while hiding the internal implementation from the outside world.**

---

# 🔐 Data Hiding

Data Hiding is achieved by declaring variables as **private**.

Example

```java
class Student{

    private String name;

}
```

Now,

```java
Student s = new Student();

s.name = "Anshika";
```

Compilation Error

```
name has private access in Student
```

The variable cannot be accessed directly from outside the class.

---

# 📖 Getter and Setter Methods

To access private variables, Java uses **Getter** and **Setter** methods.

---

## Setter Method

Used to assign values.

```java
class Student{

    private String name;

    public void setName(String name){

        this.name = name;

    }

}
```

---

## Getter Method

Used to retrieve values.

```java
class Student{

    private String name;

    public String getName(){

        return name;

    }

}
```

---

# 💻 Complete Example

```java
class Student{

    private String name;

    public void setName(String name){

        this.name = name;

    }

    public String getName(){

        return name;

    }

}

public class Main{

    public static void main(String[] args){

        Student s = new Student();

        s.setName("Anshika");

        System.out.println(s.getName());

    }

}
```

Output

```
Anshika
```

# ⚠️ Common Mistakes

## ❌ Declaring Variables as Public

Wrong

```java
public int salary;
```

Correct

```java
private int salary;
```

---

## ❌ Forgetting Validation

Wrong

```java
public void setAge(int age){

    this.age = age;

}
```

Correct

```java
public void setAge(int age){

    if(age > 0){

        this.age = age;

    }

}
```

---

## ❌ Getter Without Need

Not every private variable requires a getter.

Only expose data when necessary.

---

# 💡 Best Practices

- Keep data members private.
- Expose only required methods.
- Validate data inside setter methods.
- Avoid exposing sensitive fields.
- Follow JavaBeans naming conventions (`getX()`, `setX()`).

# 🎯 Interview Tip

### Question

What is Encapsulation?

### Answer

Encapsulation is an Object-Oriented Programming principle that binds data and methods into a single unit while restricting direct access to the data. It is achieved using private variables and public getter/setter methods, improving security, maintainability, and flexibility.
