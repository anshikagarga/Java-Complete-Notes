# ☕ Your First Java Program

Every Java journey begins with the famous **Hello, World!** program.

```java
public class HelloWorld {

    public static void main(String[] args) {

        System.out.println("Hello, World!");

    }

}
```

Output

```
Hello, World!
```

---

# 🤔 Understanding Every Line

Let's break the program into smaller pieces.

## Line 1

```java
public class HelloWorld
```

### Explanation

| Keyword | Meaning |
|----------|----------|
| public | Accessible from anywhere |
| class | Defines a class |
| HelloWorld | Name of the class |

> 💡 **Interview Tip**
>
> The filename must match the public class name.
>
> ```
> HelloWorld.java
> ```

---

## Line 2

```java
public static void main(String[] args)
```

This is called the **Main Method**.

The JVM starts executing every Java application from here.

---

### Breaking it Down

### public

Accessible to the JVM.

---

### static

Allows the JVM to call the method without creating an object.

---

### void

The method returns nothing.

---

### main

The JVM searches for this method.

---

### String[] args

Stores command-line arguments.

Example

```bash
java HelloWorld Java Python
```

Then,

```
args[0] = Java

args[1] = Python
```

---

## Line 3

```java
System.out.println("Hello, World!");
```

Prints text on the console.

---

### Understanding System.out.println()

```
System

↓

out

↓

println()
```

---

### System

A predefined Java class.

---

### out

A static object of `PrintStream`.

---

### println()

Prints data and moves the cursor to the next line.

```

```

---

```bash

```

```

```

```bash

```

The JVM reads the bytecode and executes it.

# 📖 What is Bytecode?

Bytecode is an intermediate language generated by the Java compiler.

It is **not** machine code.

It is **not** source code.

It can run on any operating system that has a JVM.

# 🧠 Components of JVM

## 1. Class Loader

Loads the `.class` file into memory.

---

## 2. Bytecode Verifier

Checks whether the bytecode is valid and secure.

---

## 3. Execution Engine

Executes the bytecode.

---

### Execution Engine has two parts

```
Interpreter

+

JIT Compiler
```

---

## Interpreter

Reads bytecode line by line.

Simple but slower.

---

## JIT Compiler

Converts frequently used bytecode into native machine code.

This improves performance.

# ⚠ Common Mistakes

## ❌ Filename and Class Name Don't Match

Wrong

```java
public class Hello {

}
```

Saved as

```
Program.java
```

Compilation Error.

Correct

```
Hello.java
```

---

## ❌ Forgetting main()

```java
class Demo{

}
```

Cannot run because there is no entry point.

---

## ❌ Missing Semicolon

```java
System.out.println("Hello")
```

Compilation Error.

---

## ❌ Wrong Case

Wrong

```java
system.out.println();
```

Correct

```java
System.out.println();
```

Java is **case-sensitive**.

---

# 💡 Best Practices

✔ Give meaningful class names.

✔ One public class per file.

✔ Follow Java naming conventions.

✔ Always format your code properly.

✔ Understand the execution flow instead of memorizing commands.

---

# 🎯 Interview Insight

Most beginners answer:

> Java is platform independent.

A stronger interview answer is:

> Java source code is compiled into platform-independent bytecode. The JVM on each operating system converts this bytecode into native machine code, allowing the same program to run on different platforms.

This shows a deeper understanding of Java's execution model.
